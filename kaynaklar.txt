OLLAMA ek bilgiler:
OLLAMA sayısı:
export OLLAMA_NUM_PARALLEL=2

# Mac terminalinde Ollama kapalıyken çalıştır
launchctl setenv OLLAMA_HOST "0.0.0.0"

# Aynı anda kaç isteğin işleneceğini belirler
export OLLAMA_NUM_PARALLEL=5

# Kaç farklı modelin bellekte hazır tutulacağını belirler
export OLLAMA_MAX_LOADED_MODELS=5

# Ollama'yı bu değişkenlerle başlatın
ollama serve

Terminalden Zorla Kapat:

Bash
pkill ollama

# Değişkenleri sistem hafızasından siliyoruz
launchctl unsetenv OLLAMA_HOST
launchctl unsetenv OLLAMA_NUM_PARALLEL
launchctl unsetenv OLLAMA_MAX_LOADED_MODELS


export OLLAMA_NUM_PARALLEL=2
export OLLAMA_MAX_LOADED_MODELS=5
launchctl setenv OLLAMA_HOST "0.0.0.0"
ollama serve




2. Linux Server (Uzak Sunucu) İçin Yapılandırma
Linux sunucuda CUDA ve cuDNN kurduğun için muhtemelen Ollama bir systemd servisi olarak çalışıyor. En profesyonel yöntem servis dosyasını güncellemektir:

Servis Düzenleme Modunu Aç:

Bash
sudo systemctl edit ollama.service
Açılan ekrana (boşsa bile) şunları yapıştır:

Ini, TOML
[Service]
Environment="OLLAMA_HOST=0.0.0.0"
Environment="OLLAMA_NUM_PARALLEL=5"
Environment="OLLAMA_MAX_LOADED_MODELS=5"
Kaydet ve Servisi Yeniden Başlat:

Bash
sudo systemctl daemon-reload
sudo systemctl restart ollama
